# 目录结构规范

本规范只定义 Skill 包“长什么样”，不讨论具体实现代码。

## 1. 最小目录骨架

```text
<skill-name>/
├── SKILL.md
├── references/
│   └── <module>/
│       └── README.md
└── scripts/
    └── <module>/
        ├── README.md
        └── <action>.py
```

## 2. 每一层的职责

| 路径 | 职责 | 必须包含什么 |
|---|---|---|
| `SKILL.md` | 总索引 | 能力概览、工作流、路由表、能力树、约束 |
| `references/<module>/README.md` | 模块说明 | 适用场景、输入要求、输出说明、动作列表 |
| `scripts/<module>/README.md` | 脚本索引 | 脚本清单、运行方式、鉴权前置条件、返回说明 |
| `scripts/<module>/<action>.py` | 唯一执行入口 | 可直接运行的 Python 脚本 |

## 3. 模块划分原则

- 一个模块只覆盖一个明确业务域
- 同一模块下的动作应该围绕同一类对象或同一条业务链路
- 如果两个动作的输入、鉴权、输出和使用场景明显不同，应拆成不同脚本
- 如果一个 Skill 只有一个业务域，也仍然保留模块层，避免后续扩展时打散

## 4. 路由与脚本映射原则

- `SKILL.md` 路由表中的每一条能力，都必须指向一个真实存在的脚本
- 模块说明中的动作列表，应覆盖该模块下全部脚本
- 脚本索引中的脚本路径，应与实际目录完全一致
- 不允许存在“文档里提到但脚本不存在”或“脚本存在但没有入口说明”的孤立文件

## 5. 目录中的禁止项

以下内容不应出现在目标 Skill 包中：

- 本地登录脚本
- 本地鉴权说明文件
- 运行过程中生成的日志、缓存、状态文件
- 与当前 Skill 无关的临时目录、测试残留和历史备份文件
- 未替换的模板占位目录

## 6. 运行时文件位置

如脚本需要记录日志、缓存或版本检查结果，统一写入工作区根目录：

```text
<workspace>/
└── .cms-log/
    ├── log/<skillcode>/
    └── state/<skillcode>/
```

运行时文件属于执行实现细节，不写回 Skill 包目录本身。
